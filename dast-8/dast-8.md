## Часть 8. Podman, покрытие и фаззинг в CI
Один из подходов к фаззингу, часто реализуемый в Golang засчет testing.F - короткие фаззинг-тесты в CI на каждый PR. При таком подходе мы жертвуем возможностью накопления большого корпуса (если не храним его отдельно) и глубиной исследования, зато приобретаем частые запуски, что может быть хорошо, например, для регрессионных-тестов.

В ходе выполнения данной части предлагается:
- написать обертку для фаззинга функции ParseImageName проекта Podman;
- собрать покрытие после фаззинга этой функции;
- настроить в SourceCraft CI, который будет запускать 5-минутный фаззинг этой функции при каждом PR.

В ходе выполнения заданий 8 части у нас должны получиться следующие артефакты работы:
- скриншоты выполнения пунктов задания;
- html-отчет о покрытии;
- публичный репозиторий в SourceCraft с настроенным CI.

### Podman
В качестве целевой функции будем рассматривать функцию ParseImageName проекта Podman - https://github.com/containers/podman/blob/main/pkg/auth/auth.go#L307. На вход она принимает строку, в которой находится закодированный в base64 JSON (рядом есть `encodeSingleAuthConfig` и примеры тестов). Для фаззинга в этом случае будем использовать testing.F-подход.
